# Pricing

## Purpose

制定既能反映价值又能市场定位的定价策略。核心是**找到"贵但值"的最优解**，而非一味低价竞争。

**适用场景：**
- 第一个产品，不知道定什么价
- 现有产品需要调整定价
- 增长遇到瓶颈，考虑提价
- 不知道怎么跟竞争对手比价

## Key Concepts

### 定价锚定公式

```
定价区间：
成本×3 ≤ 合理价格 ≤ 替代方案价格×0.7

示例（在线课程平台）：
- 成本：$10/用户（服务器+人力）
- 替代方案：线下课 $1000/学员
- 计算：$30 ≤ 价格 ≤ $700
- 建议定价：$199-399/月

核心逻辑：你给用户省了多少，就值多少
```

### 定价锚点动态调整

| 客户数量 | 价格调整 | 原因 |
|---------|---------|------|
| 0-100 | 不调整 | 建立基础，验证价值 |
| 100-500 | 重新评估 | 了解愿意付多少 |
| 500-1000 | 可小幅提价 | 证明市场接受 |
| 1000+ | 考虑分层定价 | 不同需求不同价格 |

### 定价心理學

| 策略 | 效果 | 适用场景 |
|------|------|---------|
| $99 vs $100 | 看起来便宜很多 | 标准化产品 |
| $9.99/月 vs $120/年 | 降低决策门槛 | SaaS订阅 |
| $49 + $49 vs $89 | 捆绑销售 | 多产品 |
| 基础版$29 + Pro$99 | 分层定价 | 不同需求 |

### 2. 动态定价策略

```python
# 早期定价策略
def calculate_early_pricing(metrics):
    price = 0

    if metrics.customer_count < 100:
        # 验证期：低价换取反馈
        price = metrics.cost * 5
    elif metrics.customer_count < 500:
        # 成长期：根据反馈调整
        price = metrics.cost * 8
        if metrics.nps > 50:
            price *= 1.2
    else:
        # 稳定期：利润最大化
        price = metrics.alternative_price * 0.5
        if metrics.churn < 3:
            price *= 1.1

    return price

# 提价信号
should_raise_price_signals = [
    "续费率>90%持续3个月",
    "用户主动说'不贵'",
    "经常需要限购/等位",
    "竞品比你贵但用户不流失",
    "NPS>60"
]
```

## Examples

### 成功定价案例

```
Notion 定价演进：
2018：$0-5/用户/月（免费+Starter）
2019：新增$10/用户/月的Pro
2021：团队版$15/用户/月
2024：企业版定制价格

策略：免费→低价验证→分层提价
```

### 失败定价案例

```
产品：SaaS工具
错误：$5/月低价策略
原因：认为低价=多用户

结果：
- 服务器成本无法覆盖
- 用户质量低，续费差
- 品牌定位"廉价"难以翻身

教训：低价≠增长，盈利才能持续
```

## Common Pitfalls

1. **定价过低**：害怕用户流失，不信任产品价值
2. **定价过高**：高估市场接受度
3. **忽视成本**：只考虑用户愿意付，忘了自己需要盈利
4. **价格歧视**：对不同用户定不同价（除非透明分层）
5. **不敢提价**：老客户永远用旧价格

## Decision Framework

```
定价调整检查：

提价时机：
□ 续费率>90%持续6个月
□ 用户主动表示愿意付更多
□ 成本明显上升
□ 竞品已提价
□ 供不应求

提价策略：
□ 只对新用户提价
□ 给老客户提供折扣继续
□ 提供替代方案
□ 渐进式而非突然
```

## Metrics

```
定价健康度指标：
□ 毛利率 > 70%
□ LTV/CAC > 3
□ 月收入增长率 > 10%
□ 续费率 > 85%
□ 定价符合市场定位

优化信号：
□ 问价人多，付费人少 → 价格偏高
□ 转化率高，但客单价低 → 可提价
□ 用户只买基础版 → 分层价值不够
```
