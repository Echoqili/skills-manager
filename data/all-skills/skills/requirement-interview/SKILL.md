# Requirement Interview - 需求访谈

在开始实现之前，通过结构化问卷澄清需求。

## 核心原则

1. **不盲目执行**: 收到需求后先提问，而非立即编码
2. **信息完备**: 确保所有关键维度已确认
3. **文档化**: 将澄清结果写入项目文档

## 访谈问卷

### 1. 基础信息
```
□ 需求背景是什么？
□ 目标用户是谁？
□ 验收标准是什么？
□ 截止时间？
```

### 2. 数据维度
```
□ 数据量预估是多少？（条/行）
□ 数据来源是什么？（数据库/API/文件）
□ 是否需要分页/流式处理？
□ 峰值并发量预估？
```

### 3. 格式维度
```
□ 输入格式？（CSV/Excel/JSON/数据库）
□ 输出格式？（同上 + PDF/HTML）
□ 编码要求？（UTF-8/GBK）
□ 表头/分隔符要求？
```

### 4. 权限维度
```
□ 是否需要认证？
□ 权限级别？（管理员/普通用户/访客）
□ 是否涉及敏感数据？
□ 审计日志要求？
```

### 5. 性能维度
```
□ 响应时间要求？（<1s/<5s/无要求）
□ 是否有预算限制？
□ 是否需要缓存？
□ 监控告警要求？
```

### 6. 集成维度
```
□ 需要对接哪些系统？
□ 是否有现成 SDK？
□ 接口规范？（REST/gRPC/WebSocket）
□ 错误处理策略？
```

## 输出模板

```markdown
## 需求澄清报告

### 基础信息
- 背景: [填写]
- 用户: [填写]
- 验收标准: [填写]
- 截止时间: [填写]

### 已确认事项
- [x] 数据量: < 10,000 条
- [x] 格式: CSV, UTF-8
- [x] 权限: 需要管理员权限
- [x] 性能: < 3s 响应

### 待确认事项
- [ ] 缓存策略
- [ ] 监控方案

### 风险提示
⚠️ 数据量超过 100 万条时需要分页处理
⚠️ 涉及支付需要额外安全评审
```

## 检查点

| 阶段 | 检查项 | 通过标准 |
|------|--------|----------|
| 数据 | 数据量评估 | ✅ 已确认量级 |
| 格式 | 输入输出格式 | ✅ 双方确认 |
| 权限 | 安全要求 | ✅ 已评估 |
| 性能 | SLA 要求 | ✅ 已承诺 |
| 集成 | 对接方案 | ✅ 已有方案 |

## 禁忌

- ❌ 不在未确认数据量时直接实现大数据量功能
- ❌ 不在未确认权限要求时硬编码权限逻辑
- ❌ 不在未确认格式时假设标准格式
- ❌ 不跳过任何关键维度
