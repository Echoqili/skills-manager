# 中文代码审查 (Chinese Code Review)

## 概述

中文代码审查是一种结构化的代码审查流程，专注于代码质量、可维护性和最佳实践的本地化实践。

## 工作流程

### 1. 审查准备
- [ ] 理解需求背景和业务逻辑
- [ ] 确认代码变更范围
- [ ] 了解相关技术债务

### 2. 审查维度
```
┌─────────────────────────────────────────────────────────┐
│                    代码审查清单                           │
├─────────────────────────────────────────────────────────┤
│ 1. 功能正确性    - 逻辑是否正确实现需求                    │
│ 2. 代码风格      - 是否符合团队中文命名规范                │
│ 3. 安全性        - 是否有潜在安全漏洞                      │
│ 4. 性能          - 是否有性能问题                         │
│ 5. 可测试性      - 是否容易编写单元测试                    │
│ 6. 可维护性      - 后续是否容易修改和扩展                  │
│ 7. 文档          - 复杂逻辑是否有中文注释                  │
└─────────────────────────────────────────────────────────┘
```

### 3. 审查执行
```markdown
## 审查要点

### 问题等级
- 🔴 严重: 必须修复才能合并
- 🟡 中等: 建议修复
- 🟢 轻微: 优化建议

### 反馈格式
[文件:行号] 问题描述
- 现状: ...
- 建议: ...
- 理由: ...
```

### 4. 审查反馈
- 使用中文清晰描述问题
- 提供具体的修改建议
- 解释为什么这样做更好
- 认可好的代码实践

## 中文审查特色

### 命名规范检查
- 变量/函数名是否符合中文语义
- 注释是否清晰准确
- 文档是否使用中文

### 业务逻辑审查
- 是否正确处理中文边界情况
- 是否考虑中文编码问题
- 是否处理中文特殊字符

## 审查示例

```python
# ❌ 不推荐
def calc(a, b):
    return a * b

# ✅ 推荐
def calculate_discount_price(original_price: float, discount_rate: float) -> float:
    """
    计算折扣后的价格

    Args:
        original_price: 原价（单位：元）
        discount_rate: 折扣率（如 0.8 表示 8 折）

    Returns:
        折扣后的价格
    """
    return original_price * discount_rate
```
