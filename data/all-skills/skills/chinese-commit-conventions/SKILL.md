# 中文提交规范 (Chinese Commit Conventions)

## 概述

标准化的中文提交信息规范，提高团队协作效率和代码可读性。

## 提交信息结构

```
<类型>(<影响范围>): <简短描述>

[可选的详细正文]

[可选的页脚说明]
```

## 类型标识

| 类型 | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | feat(用户): 添加手机号登录 |
| fix | 修复 bug | fix(支付): 修复重复扣费 |
| docs | 文档更新 | docs: 更新 API 文档 |
| style | 代码格式 | style: 格式化代码 |
| refactor | 重构 | refactor(订单): 重构下单逻辑 |
| perf | 性能优化 | perf: 优化查询速度 |
| test | 测试相关 | test: 添加单元测试 |
| chore | 构建/工具 | chore: 升级依赖版本 |
| revert | 撤销 | revert: 撤销上次提交 |

## 范围标识

常用范围：
- `用户`: 用户模块
- `订单`: 订单模块
- `支付`: 支付模块
- `商品`: 商品模块
- `首页`: 首页模块
- `搜索`: 搜索功能
- `缓存`: 缓存相关
- `数据库`: 数据库操作
- `API`: 接口相关
- `权限`: 权限相关

## 规范示例

### 1. 功能开发
```
feat(用户): 添加短信验证码登录功能

- 集成阿里云短信服务
- 添加验证码防刷机制
- 实现 5 分钟有效期内验证

关闭 #123
```

### 2. Bug 修复
```
fix(购物车): 修复商品数量为 0 时仍然可以下单的问题

问题原因：未校验商品数量下限
修复方案：添加数量校验，低于 1 时提示用户

关联 #456
```

### 3. 重构
```
refactor(订单服务): 重构订单状态流转逻辑

- 将状态机从 if-else 改为枚举驱动
- 添加状态转换合法性校验
- 提升代码可维护性

性能提升 30%
```

### 4. 文档更新
```
docs: 更新项目 README 文档

- 添加快速开始指南
- 补充 API 文档链接
- 更新部署说明
```

## 常用命令

```bash
# 使用 conventional commits 插件
git commit -m "feat(模块): 描述"

# 提交并执行检查
git commit -m "fix: 修复问题" --no-verify

# 修改上次提交
git commit --amend
```

## 自动化工具

### commitlint 配置
```json
{
  "rules": {
    "type-enum": [2, "always", ["feat", "fix", "docs", "style", "refactor", "perf", "test", "chore"]]
  }
}
```

### commitizen 交互式提交
```bash
# 安装
npm install -g commitizen

# 使用
git cz
```

## 好处

1. **可读性**: 清晰了解每次变更内容
2. **自动化**: 生成 CHANGELOG
3. **搜索**: 快速定位相关提交
4. **审查**: 提高代码审查效率
