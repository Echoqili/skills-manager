# Risk Precheck - 风险预检

在编码前识别并评估潜在风险，确保系统上线后稳定运行。

## 核心风险类别

### 1. 数据风险
```
□ 数据量级：< 1000 / 1万 / 10万 / 100万 / 1000万+
□ 数据增长率：平稳 / 快速增长
□ 数据分布：均匀 / 不均衡（热点数据）
□ 冷数据处理：需要 / 不需要
```

### 2. 性能风险
```
□ 并发量：< 100 / 100-1000 / 1000-10000 / 10000+
□ 响应时间要求：< 100ms / < 500ms / < 1s / 无要求
□ QPS 预估：< 10 / 10-100 / 100-1000 / 1000+
□ 峰值倍数：1x / 2x / 5x / 10x+
```

### 3. 安全风险
```
□ 认证方式：无 / Token / OAuth / LDAP
□ 数据敏感性：公开 / 内部 / 机密 / 绝密
□ 合规要求：无 / GDPR / 等保 / 金融监管
□ 审计需求：无 / 基础日志 / 完整审计
```

### 4. 可用性风险
```
□ SLA 要求：无 / 99% / 99.9% / 99.99%
□ 容灾要求：无 / 本地备份 / 异地备份
□ 降级策略：直接失败 / 优雅降级 / 功能降级
□ 监控告警：无 / 基础 / 完善
```

### 5. 集成风险
```
□ 第三方依赖：数量 / 稳定性评估
□ 超时配置：合理 / 需优化
□ 重试机制：有 / 无 / 不完善
□ 幂等性：已处理 / 未处理
```

## 风险评估矩阵

| 风险等级 | 定义 | 处理策略 | 示例 |
|----------|------|----------|------|
| 🔴 严重 | 可能导致系统崩溃或数据丢失 | 必须解决 | 大数据量无分页 |
| 🟠 高 | 可能导致性能严重下降 | 尽快解决 | 无缓存高并发 |
| 🟡 中 | 可能导致功能异常 | 计划解决 | 错误处理不完善 |
| 🟢 低 | 不影响核心功能 | 持续改进 | 日志不完整 |

## 风险检查清单

### 数据量检查
```markdown
## 数据量风险

| 检查项 | 当前值 | 阈值 | 风险 |
|--------|--------|------|------|
| 单表行数 | 50万 | 100万 | 🟡 中 |
| 单次查询返回 | 10万 | 1万 | 🔴 严重 |
| 索引大小 | 500MB | 1GB | 🟡 中 |

**行动**: 必须实现分页，禁止一次性返回超过 10000 条
```

### 权限检查
```markdown
## 权限风险

| 检查项 | 当前状态 | 要求 | 风险 |
|--------|----------|------|------|
| 认证方式 | 无 | Token | 🟡 中 |
| 管理员特权 | 有 | 需审计 | 🟠 高 |
| 数据权限 | 无 | 行级权限 | 🔴 严重 |

**行动**: 必须实现基于角色的访问控制
```

### 性能检查
```markdown
## 性能风险

| 检查项 | 当前状态 | 要求 | 风险 |
|--------|----------|------|------|
| API 响应时间 | 2s | < 500ms | 🟠 高 |
| 数据库查询 | 1.5s | < 100ms | 🔴 严重 |
| 第三方调用 | 3s | < 1s | 🟠 高 |

**行动**: 必须优化数据库查询，添加缓存
```

## 典型风险案例

### 🔴 案例1: 大数据量导出 OOM
```
风险: 一次性加载 100 万行到内存
后果: OutOfMemoryError，进程崩溃
预防: 实现流式导出/分页处理
```

### 🔴 案例2: 权限绕过
```
风险: 用户可访问他人数据
后果: 数据泄露，法律责任
预防: 实现行级权限校验
```

### 🟠 案例3: 第三方服务超时
```
风险: 支付接口超时导致订单卡住
后果: 用户体验差，投诉增加
预防: 实现超时配置和重试机制
```

## 禁忌

- ❌ 不跳过风险评估
- ❌ 不忽略用户报告的风险
- ❌ 不在风险未解决时强行上线
- ❌ 不隐藏真实风险等级
