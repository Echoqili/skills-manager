## 概述

标准化的开发分支完成流程，确保代码质量和团队协作效率。

## 分支合并流程

### 1. 同步最新代码
```bash
# 切换到主分支
git checkout dev

# 拉取最新代码
git pull origin dev

# 切换回功能分支
git checkout feature/xxx

# 变基到最新主分支
git rebase dev
```

### 2. 解决冲突
```bash
# 查看冲突文件
git status

# 解决冲突后
git add .
git rebase --continue

# 如果想放弃这次变基
git rebase --abort
```

### 3. 最终检查
```bash
# 运行测试
npm test

# 运行 lint
npm run lint

# 构建检查
npm run build

# 提交合并
git checkout dev
git merge feature/xxx
git push origin dev
```

## 提交规范

### Merge 提交信息
```bash
git merge feature/xxx -m "merge: 合并用户登录功能分支

- 添加手机号登录
- 添加短信验证码
- 添加第三方登录（微信）

关闭 #123"
```

### 打标签
```bash
git tag -a v1.0.0 -m "版本 1.0.0 - 完成用户模块"
git push origin v1.0.0
```

## 清理工作

### 1. 删除分支
```bash
# 删除本地分支
git branch -d feature/xxx

# 删除远程分支
git push origin --delete feature/xxx
```

## 注意事项

### ✅ 应该做
1. 合并前确保本地测试通过
2. 合并冲突要仔细处理
3. 合并后通知相关人员
4. 更新任务跟踪系统状态

### ❌ 不应该做
1. 不应该在深夜合并重要分支
2. 不应该忽略测试失败
3. 不应该强制推送主分支
4. 不应该在未通知的情况下合并
