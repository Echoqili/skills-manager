# 六维信息采集引擎

> 全面采集认知模式的多维度信息，确保萃取结果的完整性和准确性

## 架构概览

六维信息采集引擎是女娲技能的核心组件，通过六个专业Agent从不同角度采集目标认知模式的全方位信息。每个Agent负责一个特定维度，共同构成对人类思维模式的完整解构。

## Agent 详细规格

### 1. 著作分析Agent

**代码标识:** `Nuwa::Agents::PublicationAnalyzer`

**数据源:**
- 正式出版的书籍
- 学术论文与文章
- 公开演讲稿与访谈文字稿
- 官方博客与专栏文章

**提取内容:**

```yaml
核心论点:
  - 主要观点（1-3个）
  - 假设前提
  - 逻辑推演链

思维框架:
  - 框架名称
  - 核心原理描述
  - 适用条件
  - 使用限制

论证结构:
  - 引入方式
  - 论证路径
  - 结论形成模式

边界条件:
  - 适用领域
  - 不适用领域
  - 临界点识别
```

**输出模板:**

```markdown
## [专家姓名] 著作分析报告

### 核心论点提取
**主要观点:**
1. [观点1]
2. [观点2]
3. [观点3]

**假设前提:**
- [假设1]
- [假设2]

### 思维框架
**框架名称:** [名称]
**核心原理:** [描述]
**适用条件:** [条件]
**使用限制:** [限制]

### 论证结构分析
[结构描述]

### 验证记录
- [验证案例1]
- [验证案例2]
```

### 4. 他者视角Agent

**代码标识:** `Nuwa::Agents::ThirdPersonView`

**采集视角:**

```yaml
同事评价:
  - 决策风格描述
  - 协作特点归纳
  - 沟通模式特征

对手评价:
  - 竞争优势识别
  - 思维盲区指出
  - 策略特点分析

学生评价:
  - 教导方法归纳
  - 核心理念传递
  - 成长帮助评估

批评者评价:
  - 偏见类型识别
  - 局限范围界定
  - 盲点清单
```

**偏差校正算法:**

```python
def calibrate_bias(self, self_view, other_views):
    """
    校正自我认知偏差
    self_view: 专家自我描述
    other_views: 多方他人评价
    """
    # 计算自我-他我差异
    discrepancy = self_view.sentiment - other_views.average_sentiment

    # 识别系统性偏差类型
    bias_type = classify_bias(discrepancy)

    # 校正后的真实画像
    calibrated_view = other_views.weighted_average(
        weights=[0.3, 0.3, 0.2, 0.2]  # 同事、对手、学生、批评者
    )

    return calibrated_view, bias_type
```

**一致性评分:**

```python
def calculate_consistency(self, timeline):
    matches = sum(1 for claim, action in timeline if matches(claim, action))
    mismatches = sum(1 for claim, action in timeline if not matches(claim, action))

    consistency_score = matches / (matches + mismatches) if (matches + mismatches) > 0 else 0

    return {
        'score': consistency_score,
        'matches': matches,
        'mismatches': mismatches,
        'highlights': extract_key_moments(timeline)
    }
```

## 采集流程

### 标准采集流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        六维信息采集流程                           │
└─────────────────────────────────────────────────────────────────┘

阶段1: 启动与规划
  │
  ├─ 确定萃取目标（专家/思想家）
  ├─ 确定采集范围和时间跨度
  └─ 分配Agent资源

阶段2: 并行采集
  │
  ├─ Agent1: 著作分析 (3-5本核心著作)
  ├─ Agent2: 即兴对话 (10+小时对话素材)
  ├─ Agent3: 碎片表达 (100+条碎片化表达)
  ├─ Agent4: 他者视角 (20+份第三方评价)
  ├─ Agent5: 行为轨迹 (5年+公开记录)
  └─ Agent6: 时间线 (全周期追踪)

阶段3: 交叉验证
  │
  ├─ 横向：各维度信息交叉核对
  ├─ 纵向：时间线连贯性验证
  └─ 矛盾：识别并标记冲突点

阶段4: 整合输出
  │
  ├─ 融合各Agent输出
  ├─ 生成统一认知画像
  └─ 提交三重验证熔炉
```
